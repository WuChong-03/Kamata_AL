# 04_02_02 SSH认证

## 1. SSH认证是什么

`SSH认证` 是通过网络连接到其他电脑或服务器时，用来安全登录和操作的认证方式。

在 GitHub 中，特别是操作 `private repository` 时，经常需要使用 SSH 认证。

SSH 认证的特点是：

- 不直接使用普通密码登录
- 通信内容会被加密
- 比单纯使用 ID 和密码更安全
- 可以让 GitHub 确认“这台电脑是被允许操作仓库的电脑”

简单理解：

```text
SSH认证 = 让 GitHub 安全确认“这是我的电脑”
```

---

## 2. 公开键暗号方式

GitHub 的 SSH 认证使用的是 `公開鍵暗号方式`。

公开键暗号方式使用一对钥匙：

| 名称 | 中文 | 作用 |
|---|---|---|
| 秘密鍵 | 私钥 | 只保存在自己电脑里，不能给别人 |
| 公開鍵 | 公钥 | 注册到 GitHub 上，用来和秘密钥配对 |

可以这样理解：

```text
秘密鍵 = 自己手里的钥匙
公開鍵 = 放在 GitHub 上的锁
```

GitHub 会通过这对钥匙确认：

```text
这台电脑持有正确的秘密鍵，所以允许它操作仓库
```

注意：

```text
秘密鍵不能给别人。
GitHub 上注册的是公開鍵，不是秘密鍵。
```

---

## 3. GitHub 中 SSH 是怎么使用的

GitHub 中的 SSH 认证大致流程是：

```text
本地电脑保存秘密鍵
↓
GitHub 注册公開鍵
↓
Push / Pull / Fetch 时进行认证
↓
认证通过后允许操作仓库
```

完成 SSH 设置后，自己的所有仓库中，`Fetch`、`Pull`、`Push` 等远程操作就可以通过 SSH 认证。

特别是 `private repository`，基本需要提前设置 SSH。

---

## 4. 在 Fork 中创建认证钥匙

首先在 Fork 中创建 SSH key。

操作流程：

```text
Fork
↓
File
↓
Configure SSH Keys...
```

打开 `Configure SSH Keys` 窗口后：

```text
+
↓
New SSH Key...
```

这里会创建一组新的 SSH key，也就是：

```text
秘密鍵 + 公開鍵
```

---

## 5. 给认证钥匙命名

创建 SSH key 时，需要输入：

| 项目 | 内容 |
|---|---|
| Key File Name | 认证钥匙名称 |
| Email | 学校邮箱 |

`Key File Name` 是自己用来识别的名字，基本可以自由填写。

例如：

```text
KamataTaro
```

`Email` 输入学校邮箱。

填写后点击：

```text
Generate
```

这样 Fork 就会生成 SSH key。

---

## 6. 复制 Public Key

SSH key 创建完成后，Fork 的 `Public Key` 栏会显示公开钥匙内容。

需要复制这一段内容，然后注册到 GitHub。

操作：

```text
Public Key 栏
↓
Copy to clipboard
```

或者全选后复制也可以。

注意：

```text
复制的是 Public Key。
不要复制秘密鍵。
```

---

## 7. 在 GitHub 中注册 SSH Key

接下来在浏览器中打开 GitHub。

先确认自己已经登录 GitHub，然后点击右上角头像：

```text
头像
↓
Settings
```

这里的 `Settings` 是账号整体设置，不是某个仓库单独的设置。

进入 Settings 后，选择：

```text
SSH and GPG keys
```

然后点击：

```text
New SSH key
```

---

## 8. 添加 SSH Key

在 `Add new SSH Key` 页面中填写：

| 项目 | 内容 |
|---|---|
| Title | 自己能看懂的名字 |
| Key | 从 Fork 复制的 Public Key |

`Title` 可以写刚才的 key 名字，例如：

```text
KamataTaro
```

`Key` 栏粘贴 Fork 中复制的 `Public Key`。

然后点击：

```text
Add SSH key
```

---

## 9. 注册成功后的状态

注册成功后，GitHub 的 `SSH keys` 栏中会显示刚刚添加的认证钥匙信息。

这表示：

```text
本地电脑中有秘密鍵
GitHub 上有公開鍵
```

两边的配对已经完成。

之后在自己的仓库中进行：

```text
Fetch
Pull
Push
```

等操作时，GitHub 就可以通过 SSH 认证确认这台电脑。

---

## 10. 注意点

- `秘密鍵` 只保存在自己电脑里，不要给别人。
- GitHub 上注册的是 `公開鍵`。
- `private repository` 通常需要 SSH 认证。
- GitHub 的 `Settings` 是账号整体设置，不是仓库单独设置。
- 如果换电脑，需要在新电脑上重新创建 SSH key，并把新的 Public Key 注册到 GitHub。
- 如果 SSH 没设置好，private 仓库可能无法正常 push / pull。

---

## 11. 流程总结

```text
Fork：File → Configure SSH Keys...
↓
+ → New SSH Key...
↓
输入 Key File Name 和 Email
↓
Generate
↓
复制 Public Key
↓
GitHub：头像 → Settings
↓
SSH and GPG keys
↓
New SSH key
↓
粘贴 Public Key
↓
Add SSH key
↓
完成 SSH 认证设置
```

---

## 12. 一句话总结

SSH 认证就是让 GitHub 确认“这台电脑有正确的秘密鍵，可以操作我的仓库”。本地电脑保存 `秘密鍵`，GitHub 注册 `公開鍵`，两边配对后，private 仓库的 `Fetch / Pull / Push` 等操作就可以正常使用。
