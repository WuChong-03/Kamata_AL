# 04_02_01 作业提交流程

## 1. 作业提交的核心

AL 科目的作业提交不是直接交文件夹，而是提交 GitHub 仓库地址，并用 `Tag` 指定要判定的 commit。

这里的 `commit`、`push` 基础操作已经在 `04_02_00 Fork仓库管理` 中整理过，所以本篇从 `Tag` 和作业提交流程开始。

核心关系：

```text
commit = 保存一个版本
push = 上传到 GitHub
Tag = 指定哪一个 commit 是作业判定版本
```

---

## 2. 首先分享自己的仓库链接

在 GitHub 网站仓库页面：

```text
Code
↓
HTTPS / private 仓库需要使用 SSH
↓
复制 URL
```

然后把复制的仓库 URL 提交到指定表单。

注意：

| 仓库类型 | 地址 |
|---|---|
| public 仓库 | HTTPS 通常可以 |
| private 仓库 | 通常需要 SSH，并邀请授课担当教员 |

---

## 3. Tag

`Tag` 是 Git 中给某个特定 `commit` 打上的标签，用于 AI 判定或老师确认作业版本。

可以理解为：

```text
Tag = 告诉系统“请看这条 commit”
```

Tag 不是新的 commit，也不是给整个文件夹打标签，而是贴在某一条 commit 上。

---

## 4. 什么时候需要打 Tag

不是每一条 commit 都要打 Tag。

如果一天 commit 了很多次，只需要给真正的作业完成版本打 Tag。

例如：

```text
commit 8：修正注释
commit 7：完成 04_02 课题    ← 打 Tag
commit 6：测试移动逻辑
commit 5：整理文件
commit 4：完成 04_01 课题    ← 打 Tag
commit 3：修复错误
commit 2：添加素材
commit 1：Initial Commit
```

简单记：

```text
commit 记录过程
Tag 指定作业判定点
```

---

## 5. Tag 的创建方法

在 Fork 中：

```text
All Commits
↓
右键点击要标记的 commit
↓
New Tag...
↓
输入 Tag 名
↓
勾选 Push
↓
Create and Push
```

如果创建 Tag 时没有勾选 `Push`，Tag 可能只存在本地，GitHub 上看不到。

---

## 6. 提交顺序

最稳的课堂操作流程：

```text
完成作业
↓
Stage
↓
Commit
↓
Push
↓
在完成版 commit 上创建 Tag
↓
Push Tag
↓
提交 GitHub 仓库 URL
```

也可以先本地 commit，再本地打 Tag，最后一起 push。  
但是为了避免漏推送，课堂上更推荐先 push commit，再创建并 push Tag。

---

## 7. 作业修正后的 Tag 差し替え

如果作业被指出问题，需要修改后重新提交。

这时不能只修改代码、commit、push。  
因为旧 Tag 还指向旧 commit。

正确理解：

```text
旧 Tag 指向旧 commit
修正后的代码在新 commit
所以必须把同名 Tag 换到新 commit 上
```

---

## 8. 删除旧 Tag

如果旧 Tag 已经 push 到 GitHub，删除时要同时删除远程 Tag。

在 Fork 中删除 Tag 时勾选：

```text
Delete tag from remote repositories
```

否则会变成：

```text
本地 Tag 删除了
GitHub 上旧 Tag 还存在
```

---

## 9. 在修正后的 commit 上重新创建 Tag

删除旧 Tag 后，在新的修正 commit 上重新创建同名 Tag。

```text
选择修正后的 commit
↓
右键
↓
New Tag...
↓
输入和之前相同的 Tag 名
↓
勾选 Push
↓
Create and Push
```

这样 GitHub 上的 Tag 才会指向新的修正版本。

---

## 10. 修正后重新提交流程

```text
收到指摘
↓
打开原来的项目文件夹
↓
修改作业
↓
确认运行
↓
Stage
↓
Commit 修正内容
↓
Push
↓
删除旧 Tag
↓
勾选 Delete tag from remote repositories
↓
在新的修正 commit 上重新创建同名 Tag
↓
勾选 Push
↓
Create and Push
```

一句话：

```text
不是改旧 commit，而是新建修正 commit，再把 Tag 移到新的 commit 上。
```

---

## 11. ex 与课题顺序的补充

`ex` 是扩展内容，课堂上可以根据担当教员的指示跳过。

如果做了 `ex`，完成后继续进入下一个课题。

例如：

```text
03_08
↓
03_08_ex1
↓
03_09
```

不建议为了配合课题编号而强行倒回去修改旧内容。

尽量接在最新 commit 后继续实现。

---

## 12. 一句话总结

作业提交时，先完成作业并 `commit → push`，再给真正用于判定的 commit 打 `Tag`。如果作业修正了，要重新 commit / push，然后删除旧 Tag，并在新的修正 commit 上重新创建同名 Tag。
