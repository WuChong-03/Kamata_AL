# namespace（命名空间）

在 C++ 中，`::` 叫做 **作用域解析运算符**，作用是告诉编译器“这个函数或内容属于谁”。

例如：

```cpp
KamataEngine::Update();
Enemy::Update();
std::cout;
```

虽然都用了 `::`，但前面的名字不一定是同一种东西。

---

## 1. namespace 的情况

例如：

```cpp
KamataEngine::Update();
Novice::DrawBox();
std::cout;
```

这里的 `KamataEngine`、`Novice`、`std` 通常都是库里的 namespace（命名空间），用于区分不同库中的函数或功能，避免同名函数冲突。

例如：

```cpp
KamataEngine::Update();
Novice::Update();
```

虽然都叫 `Update()`，但一个来自 `KamataEngine`，一个来自 `Novice`。

---

## 2. 类的情况

例如：

```cpp
Enemy::Update();
Player::Move();
GameScene::Draw();
```

这里的 `Enemy`、`Player`、`GameScene` 通常是类名，表示这个函数属于哪个类。

例如：

```cpp
void Enemy::Update() {

}
```

这表示正在实现 `Enemy` 类里的 `Update()` 函数。

---

## 3. using namespace

如果每次都写 `KamataEngine::Update()`、`KamataEngine::Initialize()`、`KamataEngine::Finalize()` 会比较麻烦，因此可以使用 `using namespace KamataEngine;` 来省略 `KamataEngine::`。

例如：

```cpp
KamataEngine::Update(); // 不使用 using namespace

Update(); // 使用 using namespace 后
```

---

## 4. using namespace 的注意点

不要乱写多个 `using namespace`，否则可能发生命名冲突。

例如：

```cpp
using namespace KamataEngine;
using namespace Novice;
```

如果两个库里刚好都有 `Update()`，编译器可能无法判断你要调用哪一个。

不要在 `.h` 头文件里写 `using namespace`，因为头文件会被很多 `.cpp` include，容易污染整个项目。通常只在 `.cpp` 文件中使用。

---

## 5. 总结

| 写法 | 类型 | 含义 |
|---|---|---|
| `KamataEngine::Update()` | namespace | `KamataEngine` 里的 `Update()` |
| `Novice::DrawBox()` | namespace | `Novice` 里的 `DrawBox()` |
| `std::cout` | namespace | `std` 里的 `cout` |
| `Enemy::Update()` | 类 | `Enemy` 类里的 `Update()` |
| `Player::Move()` | 类 | `Player` 类里的 `Move()` |
