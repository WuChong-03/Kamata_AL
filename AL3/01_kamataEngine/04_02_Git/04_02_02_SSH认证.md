# 04_02_02 SSH认证

SSH认证可以理解成：

**给 GitHub 仓库加了一把锁**

SSH 会生成：

- **公開鍵** : 上传到 GitHub 的锁
- **秘密鍵** : 保存在自己电脑里的钥匙

和 GitHub账户密码不一样，SSH 主要用于：

**确认“这是不是被信任的设备”**

也就是说：

即使有人拥有某些手段，尝试绕过普通登录直接进行 Git 操作，GitHub 也会继续检查：

**“这台设备有没有正确的秘密鍵”**

所以从安全层级来说：

- GitHub账号密码 = 第一层身份认证
- SSH = 第二层设备认证

SSH 的核心思想是：

即使你不通过普通网页登录，GitHub 也会继续检查：

**“这是不是被信任的电脑”**

例如：

- Push
- Pull
- Fetch

这些 Git 操作时：

GitHub 会检查：

**这台电脑有没有对应的秘密鍵**

如果：

**没有正确秘密鍵**

那么：

**SSH认证失败**

当然：

如果 GitHub账号本身已经完全被盗，攻击者甚至可以修改 SSH key，那么 SSH 也会失去意义。

所以真正完整的安全体系应该是：

- GitHub密码
- 2FA
- SSH

##  public 仓库、private 仓库、collaborator 的关系

### public 仓库

`public repository` 是公开仓库,别人看得见,操作需要协作者权限

当然 public 仓库可以用也可以不用 SSH,看你心情

---

### private 仓库

`private repository` 是私人仓库,别人看不到,操作需要协作者权限

private 仓库通常：更常使用 SSH

---

### collaborator

`collaborator`协作者

如果一个仓库是 private：只有被邀请的协作者才能访问。

## 6. 多人开发时 SSH 的关系

多人开发时：

```text
不是所有人共用一把秘密鍵
```

而是：

```text
每个人
都有自己的 SSH key
```

例如：

```text
你：
  吴冲-PC-key

老师：
  teacher-note-key

队友：
  tanaka-desktop-key
```

GitHub 会同时检查：

```text
1. GitHub账号有没有仓库权限
2. 这台电脑有没有对应秘密鍵
```

两边都通过：

```text
才能 Push / Pull
```

所以：

```text
collaborator
=
仓库权限

SSH
=
设备认证
```

---

## 7. SSH key 是给谁用的

SSH key：

```text
不是给某个仓库单独使用
```

而是：

```text
给“这台电脑”使用
```

通常情况：

```text
一个人
+
一台电脑
+
一把 SSH key
```

就够了。

这一把 key 可以操作：

```text
多个仓库
多个 private repository
多个团队项目
```

只要：

```text
GitHub账号拥有这些仓库权限
```

即可。

---

## 8. GitHub 中 SSH 是怎么使用的

GitHub 中的 SSH 认证流程：

```text
本地电脑保存秘密鍵
↓
GitHub 注册公開鍵
↓
Push / Pull / Fetch 时进行认证
↓
认证通过后允许操作仓库
```

完成 SSH 设置后：

```text
如果仓库 remote 地址使用 SSH 地址
```

就会通过 SSH 认证进行远程操作。

SSH 地址通常长这样：

```text
git@github.com:用户名/仓库名.git
```

如果 remote 地址还是：

```text
https://github.com/用户名/仓库名.git
```

那么：

```text
即使设置了 SSH key
这个仓库当前也不一定会走 SSH 认证
```

## 9. 在 Fork 中创建 SSH key

首先在 Fork 中创建 SSH key。

操作流程：

```text
Fork
↓
File
↓
Configure SSH Keys...
```

打开 `Configure SSH Keys` 后：

```text
+
↓
New SSH Key...
```

这里会生成：

```text
秘密鍵 + 公開鍵
```

---

## 10. 给 SSH key 命名

创建 SSH key 时，需要输入：

| 项目 | 内容 |
|---|---|
| Key File Name | 自己能看懂的名字 |
| Email | 学校邮箱 / GitHub邮箱 |

例如：

```text
KamataTaro
```

填写后点击：

```text
Generate
```

Fork 就会生成 SSH key。

---

## 11. 复制 Public Key

SSH key 创建完成后：

```text
Fork 的 Public Key 栏
```

会显示公開鍵内容。

操作：

```text
Public Key
↓
Copy to clipboard
```

注意：

```text
复制的是 Public Key
不要复制秘密鍵
```

---

## 12. 在 GitHub 中注册 SSH key

打开 GitHub：

```text
头像
↓
Settings
```

注意：

```text
这里是 GitHub账号整体设置
不是某个仓库设置
```

进入：

```text
SSH and GPG keys
↓
New SSH key
```

---

## 13. 添加 SSH key

在 `Add new SSH Key` 页面：

| 项目 | 内容 |
|---|---|
| Title | 自己能看懂的名字 |
| Key | 从 Fork 复制的 Public Key |

然后：

```text
Add SSH key
```

---

## 14. 注册成功后的状态

注册成功后：

```text
本地电脑中有秘密鍵
GitHub 上有公開鍵
```

两边的配对已经完成。

之后如果仓库使用 SSH remote 地址：

```text
Fetch
Pull
Push
```

等操作时：

```text
GitHub 会检查：
“这台电脑有没有正确的秘密鍵”
```

验证成功：

```text
允许操作仓库
```
