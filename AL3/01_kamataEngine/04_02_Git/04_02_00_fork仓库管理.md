# 04_02_00 Fork仓库管理

## 1. Fork 相关操作

### 1.1 Fork 中创建本地仓库

```text
File
↓
Init New Repository...
↓
选择项目根文件夹
↓
创建本地仓库
```

这里选择的文件夹是项目的根文件夹，也就是包含 `.gitignore`、项目文件夹、资源文件夹等内容的最外层文件夹。

创建后会生成 `.git` 文件夹。`.git` 是 Git 的管理文件夹，一般不要手动修改。

---

### 1.2 初始提交 Initial Commit

创建本地仓库之后，暂时还没有 commit 历史，需要先做一个初始提交。

课件中建议先只选择 `.gitignore` 进行第一次 commit。

```text
Local Changes
↓
从 Unstaged 中选择 .gitignore
↓
Stage
↓
输入 commit message：Initial Commit
↓
Commit
```

---

### 1.3 Staged 和 Unstaged

| 状态 | 含义 |
|---|---|
| Unstaged | 还没有放入本次提交的修改 |
| Staged | 已经准备放入本次提交的修改 |

`Stage` 的意思是：把文件加入下一次 commit 的候选内容。

---

### 1.4 Initial Commit 后的结果

完成 Initial Commit 后，可以在 `All Commits` 中看到提交记录。

此时通常只有一个 commit。之后再继续把其他文件 `stage → commit → push`。

---

## 2. 本地仓库和远程仓库连接

本地仓库创建后，还需要和 GitHub 上的远程仓库连接。

这一步叫设置 `remote`。

### 2.1 在 Fork 中添加 Remote

```text
Remotes
↓
右键
↓
Add New Remote...
```

然后在 GitHub 仓库页面复制远程仓库地址，并粘贴到 Fork 中。

---

### 2.2 SSH 与 HTTPS

GitHub 上可以复制两种地址：

| 地址类型 | 说明 |
|---|---|
| HTTPS | public 仓库通常可以使用 |
| SSH | private 仓库通常需要使用 |

如果是 `private` 仓库，通常还需要完成 SSH 认证，并邀请授课担当教员作为 collaborator。

---

## 3. Push：上传到 GitHub

连接 remote 后，就可以 push。

`Push` 的意思是：

```text
把本地 commit 上传到远程仓库 GitHub
```

Fork 中的基本流程：

```text
Push
↓
确认 branch 是 master
↓
确认推送目标是 origin/master
↓
Push
```

Push 选项通常保持默认即可。

如果勾选 `Push all tags`，Tag 也会一起推送到 GitHub。

---

## 4. Commit 与 Push 的关系

### 4.1 Commit

`commit` 是保存到本地仓库。

```text
commit = 保存一次本地修改记录
```

commit 后，本地 Git 历史会更新，但 GitHub 上还不一定有。

---

### 4.2 Push

`push` 是上传到远程仓库。

```text
push = 把本地 commit 上传到 GitHub
```

如果只 commit 不 push，老师在 GitHub 上看不到你的最新修改。

---

### 4.3 简单理解

```text
commit 是存到自己电脑
push 是传到 GitHub
```

---

## 5. Visual Studio 与 Fork 的配合

Visual Studio 和 Fork 可以并用。

课程建议：

- 继续在 Visual Studio 中写代码
- 可以在 Visual Studio 中 commit / push
- Git 的整体状态、分支树、提交历史建议在 Fork 中确认

原因是团队制作时分支可能会变复杂，Fork 的 Git tree 更直观。

简单分工：

| 工具 | 主要用途 |
|---|---|
| Visual Studio | 写代码、运行项目 |
| Fork | 查看 commit、branch、tag、remote |
| GitHub | 远程保存、课题提交、老师确认 |

---

## 6. 最重要的流程总结

### 6.1 初次创建项目流程

```text
准备项目文件夹
↓
Fork：Init New Repository
↓
生成 .git
↓
Stage .gitignore
↓
Initial Commit
↓
GitHub 创建 Private Repository
↓
Fork 添加 Remote
↓
Push
```

---

### 6.2 平时开发流程

```text
master 分支上编辑
↓
完成一个小任务
↓
stage
↓
commit
↓
push
```

---

## 7. 一句话总结

Fork 仓库管理的重点是：先用 Fork 创建本地仓库，再做 Initial Commit，然后在 GitHub 创建远程仓库，通过 Remote 连接两边，最后用 Push 把本地 commit 上传到 GitHub。
